\doxysubsection{Detailed Description}
A layer that may contain objects, to control the rendering order. 

The Layering system of \doxylink{class_q_custom_plot}{QCustom\+Plot} is the mechanism to control the rendering order of the elements inside the plot.

It is based on the two classes \doxylink{class_q_c_p_layer}{QCPLayer} and \doxylink{class_q_c_p_layerable}{QCPLayerable}. \doxylink{class_q_custom_plot}{QCustom\+Plot} holds an ordered list of one or more instances of \doxylink{class_q_c_p_layer}{QCPLayer} (see \doxylink{class_q_custom_plot_a8f33108881c2e3c2fec50aa321e169af}{QCustom\+Plot\+::add\+Layer}, \doxylink{class_q_custom_plot_a0a96244e7773b242ef23c32b7bdfb159}{QCustom\+Plot\+::layer}, \doxylink{class_q_custom_plot_ae896140beff19424e9e9e02d6e331104}{QCustom\+Plot\+::move\+Layer}, etc.). When replotting, \doxylink{class_q_custom_plot}{QCustom\+Plot} goes through the list of layers bottom to top and successively draws the layerables of the layers into the paint buffer(s).

A \doxylink{class_q_c_p_layer}{QCPLayer} contains an ordered list of \doxylink{class_q_c_p_layerable}{QCPLayerable} instances. \doxylink{class_q_c_p_layerable}{QCPLayerable} is an abstract base class from which almost all visible objects derive, like axes, grids, graphs, items, etc.\hypertarget{class_q_c_p_layer_qcplayer-defaultlayers}{}\doxysubsection{\texorpdfstring{Default layers}{Default layers}}\label{class_q_c_p_layer_qcplayer-defaultlayers}
Initially, \doxylink{class_q_custom_plot}{QCustom\+Plot} has six layers\+: "{}background"{}, "{}grid"{}, "{}main"{}, "{}axes"{}, "{}legend"{} and "{}overlay"{} (in that order). On top is the "{}overlay"{} layer, which only contains the \doxylink{class_q_custom_plot}{QCustom\+Plot}\textquotesingle{}s selection rect (\doxylink{class_q_custom_plot_ad7df2bcbba307e644db383b449e31efd}{QCustom\+Plot\+::selection\+Rect}). The next two layers "{}axes"{} and "{}legend"{} contain the default axes and legend, so they will be drawn above plottables. In the middle, there is the "{}main"{} layer. It is initially empty and set as the current layer (see \doxylink{class_q_custom_plot_a73a6dc47c653bb6f8f030abca5a11852}{QCustom\+Plot\+::set\+Current\+Layer}). This means, all new plottables, items etc. are created on this layer by default. Then comes the "{}grid"{} layer which contains the \doxylink{class_q_c_p_grid}{QCPGrid} instances (which belong tightly to \doxylink{class_q_c_p_axis}{QCPAxis}, see \doxylink{class_q_c_p_axis_a63f1dd2df663680d2a8d06c19592dd63}{QCPAxis\+::grid}). The Axis rect background shall be drawn behind everything else, thus the default \doxylink{class_q_c_p_axis_rect}{QCPAxis\+Rect} instance is placed on the "{}background"{} layer. Of course, the layer affiliation of the individual objects can be changed as required (\doxylink{class_q_c_p_layerable_ab0d0da6d2de45a118886d2c8e16d5a54}{QCPLayerable\+::set\+Layer}).\hypertarget{class_q_c_p_layer_qcplayer-ordering}{}\doxysubsection{\texorpdfstring{Controlling the rendering order via layers}{Controlling the rendering order via layers}}\label{class_q_c_p_layer_qcplayer-ordering}
Controlling the ordering of layerables in the plot is easy\+: Create a new layer in the position you want the layerable to be in, e.\+g. above "{}main"{}, with \doxylink{class_q_custom_plot_a8f33108881c2e3c2fec50aa321e169af}{QCustom\+Plot\+::add\+Layer}. Then set the current layer with \doxylink{class_q_custom_plot_a73a6dc47c653bb6f8f030abca5a11852}{QCustom\+Plot\+::set\+Current\+Layer} to that new layer and finally create the objects normally. They will be placed on the new layer automatically, due to the current layer setting. Alternatively you could have also ignored the current layer setting and just moved the objects with \doxylink{class_q_c_p_layerable_ab0d0da6d2de45a118886d2c8e16d5a54}{QCPLayerable\+::set\+Layer} to the desired layer after creating them.

It is also possible to move whole layers. For example, If you want the grid to be shown in front of all plottables/items on the "{}main"{} layer, just move it above "{}main"{} with \doxylink{class_q_custom_plot_ae896140beff19424e9e9e02d6e331104}{QCustom\+Plot\+::move\+Layer}.

The rendering order within one layer is simply by order of creation or insertion. The item created last (or added last to the layer), is drawn on top of all other objects on that layer.

When a layer is deleted, the objects on it are not deleted with it, but fall on the layer below the deleted layer, see \doxylink{class_q_custom_plot_a40f75e342c5eaab6a86066a42a0e2a94}{QCustom\+Plot\+::remove\+Layer}.\hypertarget{class_q_c_p_layer_qcplayer-buffering}{}\doxysubsection{\texorpdfstring{Replotting only a specific layer}{Replotting only a specific layer}}\label{class_q_c_p_layer_qcplayer-buffering}
If the layer mode (\doxylink{class_q_c_p_layer_a938d57b04f4e4c23cedf1711f983919b}{set\+Mode}) is set to \doxylink{class_q_c_p_layer_a67dcfc1590be2a1f2227c5a39bb59c7cada274b1644a2a3c1b794c052f1601bb2}{lm\+Buffered}, you can replot only this specific layer by calling \doxylink{class_q_c_p_layer_adefd53b6db02f470151c416f42e37180}{replot}. In certain situations this can provide better replot performance, compared with a full replot of all layers. Upon creation of a new layer, the layer mode is initialized to \doxylink{class_q_c_p_layer_a67dcfc1590be2a1f2227c5a39bb59c7ca6294b5ed9040aede762c626f5108f934}{lm\+Logical}. The only layer that is set to \doxylink{class_q_c_p_layer_a67dcfc1590be2a1f2227c5a39bb59c7cada274b1644a2a3c1b794c052f1601bb2}{lm\+Buffered} in a new \doxylink{class_q_custom_plot}{QCustom\+Plot} instance is the "{}overlay"{} layer, containing the selection rect.

void QCPLayer\+::set\+Mode (\begin{DoxyParamCaption}\item[{\mbox{\hyperlink{class_q_c_p_layer_a67dcfc1590be2a1f2227c5a39bb59c7c}{QCPLayer\+::\+Layer\+Mode}}}]{mode}{}\end{DoxyParamCaption})}

Sets the rendering mode of this layer.

If {\itshape mode} is set to \doxylink{class_q_c_p_layer_a67dcfc1590be2a1f2227c5a39bb59c7cada274b1644a2a3c1b794c052f1601bb2}{lm\+Buffered} for a layer, it will be given a dedicated paint buffer by the parent \doxylink{class_q_custom_plot}{QCustom\+Plot} instance. This means it may be replotted individually by calling \doxylink{class_q_c_p_layer_adefd53b6db02f470151c416f42e37180}{QCPLayer\+::replot}, without needing to replot all other layers.

Layers which are set to \doxylink{class_q_c_p_layer_a67dcfc1590be2a1f2227c5a39bb59c7ca6294b5ed9040aede762c626f5108f934}{lm\+Logical} (the default) are used only to define the rendering order and can\textquotesingle{}t be replotted individually.

Note that each layer which is set to \doxylink{class_q_c_p_layer_a67dcfc1590be2a1f2227c5a39bb59c7cada274b1644a2a3c1b794c052f1601bb2}{lm\+Buffered} requires additional paint buffers for the layers below, above and for the layer itself. This increases the memory consumption and (slightly) decreases the repainting speed because multiple paint buffers need to be joined. So you should carefully choose which layers benefit from having their own paint buffer. A typical example would be a layer which contains certain layerables (e.\+g. items) that need to be changed and thus replotted regularly, while all other layerables on other layers stay static. By default, only the topmost layer called "{}overlay"{} is in mode \doxylink{class_q_c_p_layer_a67dcfc1590be2a1f2227c5a39bb59c7cada274b1644a2a3c1b794c052f1601bb2}{lm\+Buffered}, and contains the selection rect.
